## Why

We want the `ozi-rs` UI to feel as polished as modern shadcn-style applications (e.g. meetily) while keeping Svelte 5 and our four Catppuccin flavours. Today the frontend has no design-token contract, no primitive library, no `cn()` helper, no form runtime, and no Tailwind. The current `theme.ts` only emits `--ctp-*` hex variables, which the shadcn-svelte ecosystem cannot consume — its primitives require HSL-triplet semantic tokens (`--background`, `--foreground`, `--primary`, …) and a `class="dark"` toggle.

This change introduces the design-token contract and the shadcn-svelte primitive library on top of the SvelteKit foundation. It is **Change 2 of 3** in the shadcn UI-kit migration described in `docs/superpowers/specs/2026-05-17-shadcn-ui-kit-svelte-design.md` — see decisions D1, D4, D5, D6, D7, D8 and the "Catppuccin → semantic token mapping" architecture section.

**Dependency:** Change 1 (`migrate-to-sveltekit`) MUST land first. This change assumes the SvelteKit shell, `$lib` alias, `routes/` layout, ESLint flat config, and Prettier config from Change 1 are already present. This change adds `prettier-plugin-tailwindcss` to that Prettier config; it does not establish the config itself.

## What Changes

- Install Tailwind v4 + `autoprefixer` + `postcss` + `tailwindcss-animate` + `@tailwindcss/typography`; wire `tailwind.config.ts` with a token-aware colour scale using `hsl(var(--background) / <alpha-value>)`.
- Add `prettier-plugin-tailwindcss` to the Prettier config from Change 1.
- Extend `src/lib/theme.ts` with `SEMANTIC_MAP_LIGHT`, `SEMANTIC_MAP_DARK`, and `applySemanticTokens(flavour)`. Keep the existing `--ctp-*` palette layer; emit semantic tokens alongside it. Toggle `class="dark"` on `<html>` whenever the active flavour is not Latte.
- Add `src/lib/utils.ts` exporting `cn() = twMerge(clsx(...))`.
- Run `shadcn-svelte init`; install primitives: button, dialog, popover, select, tabs, tooltip, switch, separator, scroll-area, input, label, sonner, slider, card, table.
- Install `lucide-svelte`, `svelte-sonner`, `tailwind-variants`.
- Install `felte` + `zod`; add `src/lib/forms/create-form.ts` (felte + zod resolver) and a placeholder `src/lib/forms/schemas/` directory.
- Create `THIRD_PARTY_LICENSES.md`; update `tauri.conf.json` `licenseFile`; add a Credits section to README.
- Vitest tests for theme mapping across all four Catppuccin flavours (verify HSL-triplet shape `"H S% L%"`).
- `@testing-library/svelte` smoke tests for every installed primitive.

**Explicitly NOT in this change:** migration of the existing nine components — `Console`, `ThemePicker`, `SymbolPicker`, `SimplifyPanel`, `TracksPanel`, `WaypointsPanel`, `TrackPointsPanel`, `Sidebar`, `MapView`. They keep their current `<style>` blocks; Change 3 migrates them.

**Explicitly NOT adopted:** `mode-watcher` — decision D5 keeps the four-flavour `theme.ts` as the source of truth for light/dark.

## Impact

- Affected capabilities:
  - `ui-shell` — MODIFIED: theme requirement extended to emit semantic HSL-triplet tokens alongside `--ctp-*`. ADDED: shadcn-svelte primitive library is the source of UI primitives; `cn()` helper available from `$lib/utils`; `felte` + `zod` is the form runtime; track and waypoint colours are isolated from the theme system.
- Affected code (this change's scope only):
  - `package.json`, `pnpm-lock.yaml` — new dependencies.
  - `tailwind.config.ts`, `postcss.config.js`, `src/app.css` — Tailwind setup.
  - `.prettierrc` (from Change 1) — `prettier-plugin-tailwindcss` added.
  - `src/lib/theme.ts` — extended (existing `applyTheme` / `applyStoredTheme` kept).
  - `src/lib/utils.ts` — new.
  - `src/lib/components/ui/**` — generated by `shadcn-svelte init` + per-primitive add.
  - `src/lib/forms/create-form.ts`, `src/lib/forms/schemas/.gitkeep` — new.
  - `THIRD_PARTY_LICENSES.md` — new.
  - `tauri.conf.json` — `licenseFile` pointer.
  - `README.md` — Credits section.
  - `src/test/theme.test.ts`, `src/test/ui/<primitive>.test.ts` — new tests.
- Existing user-visible behaviour: unchanged. Theme picker continues to switch the four flavours; no component looks different yet (Change 3 does that).
- Project file format: no change.
- Track/waypoint colour storage: unchanged — domain RGBA stays the single source of truth (D6).
